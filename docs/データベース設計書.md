# データベース設計書 (Database Design Document)

## 1. 概要
本ドキュメントは、B2B売掛金・債権管理システム（RCMS）のデータベース構造を定義する。

## 2. テーブル一覧

| テーブル名 | 論理名 | 説明 |
| :--- | :--- | :--- |
| `users` | ユーザー | システムを利用する従業員情報。 |
| `companies` | 取引先企業 | 売掛金の対象となる顧客企業情報。 |
| `invoices` | 請求書 | 売掛金（請求）の基本データとステータス。 |
| `reconciliations` | 入金消込 | 銀行取引と請求書の照合・消込履歴。 |

---

## 3. テーブル詳細

### 3.1 `users` (ユーザー)
システム利用者の認証および権限情報を管理する。

| カラム名 | データ型 | 必須 | キー | 説明 |
| :--- | :--- | :---: | :---: | :--- |
| `id` | bigint | Yes | PK | 内部ID (Auto Increment) |
| `username` | varchar(50) | Yes | UK | ユーザー名（ログインID） |
| `password_hash` | varchar(255) | Yes | - | ハッシュ化されたパスワード |
| `real_name` | varchar(100) | Yes | - | 氏名（実名） |
| `role` | enum | Yes | - | ロール (`ADMIN`, `SALES`, `LEGAL`, `FINANCE`) |
| `status` | tinyint | Yes | - | 状態 (1:有効, 0:無効) |
| `created_at` | datetime | Yes | - | 作成日時 |
| `updated_at` | datetime | Yes | - | 更新日時 |

### 3.2 `companies` (取引先企業)
取引先のマスタ情報および与信情報を管理する。

| カラム名 | データ型 | 必須 | キー | 説明 |
| :--- | :--- | :---: | :---: | :--- |
| `id` | bigint | Yes | PK | 内部ID (Auto Increment) |
| `company_code` | varchar(50) | Yes | UK | 企業コード |
| `company_name` | varchar(255) | Yes | - | 企業名 |
| `credit_rating` | varchar(10) | No | - | 信用格付け |
| `credit_limit` | decimal(18) | No | - | 与信限度額 |
| `pic_username` | varchar(50) | Yes | - | 担当者ユーザー名 |
| `address` | varchar(255) | Yes | - | 住所 |
| `email` | varchar(255) | No | - | メールアドレス |
| `phone_num` | varchar(50) | No | - | 電話番号 |
| `fax_num` | varchar(50) | No | - | FAX番号 |
| `contract_id` | varchar(100) | No | - | 契約番号 |
| `created_at` | datetime | Yes | - | 作成日時 |
| `updated_at` | datetime | Yes | - | 更新日時 |

### 3.3 `invoices` (請求書)
個別の請求データおよび債権の状態を管理する。

| カラム名 | データ型 | 必須 | キー | 説明 |
| :--- | :--- | :---: | :---: | :--- |
| `id` | bigint | Yes | PK | 内部ID (Auto Increment) |
| `invoice_no` | varchar(50) | Yes | UK | 請求書番号 |
| `company_code` | varchar(50) | Yes | - | 企業コード |
| `invoice_amount` | decimal(18) | Yes | - | 請求総額（本金 + 利息） |
| `issue_date` | date | Yes | - | 発行日 |
| `due_date` | date | Yes | - | 支払期限 |
| `created_by` | varchar(50) | Yes | - | 作成者ユーザー名 |
| `status` | enum | Yes | - | ステータス (`NORMAL`, `PAID`, `OVERDUE`, `DUNNING`, `LITIGATION`) |
| `principal_amount` | decimal(18) | Yes | - | 本金金額 |
| `interest_amount` | decimal(18) | No | - | 遅延損害金累計 |
| `interest_start_date` | date | No | - | 起息日（遅延損害金計算開始日） |
| `notes` | text | No | - | 備考 |
| `created_at` | datetime | Yes | - | 作成日時 |
| `updated_at` | datetime | Yes | - | 更新日時 |

### 3.4 `reconciliations` (入金消込)
銀行振込データと請求書の紐付け履歴を管理する。

| カラム名 | データ型 | 必須 | キー | 説明 |
| :--- | :--- | :---: | :---: | :--- |
| `id` | bigint | Yes | PK | 内部ID (Auto Increment) |
| `invoice_no` | varchar(50) | Yes | - | 請求書番号 |
| `bank_transaction_id` | varchar(50) | Yes | - | 銀行取引ID |
| `payer_name` | varchar(50) | No | - | 振込依頼人名 |
| `transaction_date` | date | No | - | 取引日 |
| `transaction_amount` | decimal(18) | Yes | - | 取引金額 |
| `reconciliation_amount` | decimal(18) | No | - | 消込適用金額 |
| `match_type` | enum | Yes | - | 照合タイプ (`EXACT`, `NAME_FUZZY`, `AMOUNT_VARIANCE`, `COMBINED`) |
| `variance_amount` | decimal(18) | No | - | 差異金額 |
| `description` | text | No | - | 説明・備考 |
| `reconciled_at` | datetime | Yes | - | 消込実行日時 |
| `reconciled_by` | varchar(50) | No | - | 実行者（システムまたはユーザー名） |
