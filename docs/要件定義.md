# 1. システム概要・目的・対象範囲

## 1.1 システム概要

本システムは、中堅商社におけるB2B取引の売掛金管理業務を一元化し、取引先管理、請求書管理、入金管理、自動入金消込、バッチ処理、ダッシュボード機能を提供するWebシステムである。

従来、売掛金管理業務はExcel、会計システム、銀行入金データ、担当者ごとの管理資料などに分散して実施されており、請求状況、入金状況、未消込残高、滞留債権の把握に時間を要している。

本システムの導入により、売掛金に関する情報を一元管理し、入金消込作業の自動化、請求・回収状況の可視化、業務効率化、債権管理精度の向上を実現する。

---

## 1.2 システム導入の目的

本システムの主な導入目的は以下の通りとする。

| No. | 目的              | 内容                                        |
| --- | --------------- | ----------------------------------------- |
| 1   | 売掛金業務の一元管理      | 取引先、請求、入金、消込、残高、滞留債権をシステム上で一元的に管理する。      |
| 2   | 入金消込業務の効率化      | 銀行入金データと請求データを照合し、入金消込作業の自動化・半自動化を実現する。   |
| 3   | 債権状況の可視化        | 未入金、消込済、未消込、期限超過、滞留債権などの状況をダッシュボードで可視化する。 |
| 4   | 人的ミスの削減         | Excel管理や手作業による転記ミス、消込漏れ、請求漏れを削減する。        |
| 5   | 月次締め処理の迅速化      | 月末時点の売掛残高、入金状況、未消込明細を迅速に集計できるようにする。       |
| 6   | 管理部門・営業部門間の情報共有 | 経理部門、営業部門、管理者が同一データを参照できる環境を整備する。         |

---

## 1.3 対象業務範囲

本システムの対象業務範囲は以下とする。

| 業務領域     | 対象内容                            |
| -------- | ------------------------------- |
| 取引先管理    | 取引先情報、請求先情報、支払条件、与信情報、担当営業情報の管理 |
| 請求書管理    | 請求書データの登録、参照、検索、ステータス管理、請求残高管理  |
| 入金管理     | 銀行入金データの取込、入金明細の管理、未消込入金の管理     |
| 自動入金消込   | 請求データと入金データの自動照合、消込候補の提示、手動消込   |
| バッチ処理    | 銀行データ取込、請求残高更新、滞留債権判定、月次集計処理    |
| ダッシュボード  | 売掛残高、未入金金額、滞留債権、消込状況、入金予定の可視化   |
| 権限管理     | 経理担当者、営業担当者、管理者などの利用権限管理        |
| 帳票・CSV出力 | 売掛残高一覧、未入金一覧、消込結果一覧、滞留債権一覧の出力   |

---

## 1.4 対象外業務範囲

本システムでは、以下の業務は原則として対象外とする。

| No. | 対象外業務      | 備考                                      |
| --- | ---------- | --------------------------------------- |
| 1   | 会計仕訳の本登録   | 会計システム側で実施する想定。本システムでは必要に応じて連携データを出力する。 |
| 2   | 請求書の電子送付   | 初期スコープでは対象外。将来的な拡張機能として検討可能。            |
| 3   | 与信審査ワークフロー | 与信金額の参照・管理は対象とするが、承認ワークフローは別途検討とする。     |
| 4   | 督促状の自動送付   | 滞留債権の抽出は対象とするが、督促メール・督促状の自動送付は初期対象外とする。 |
| 5   | 支払管理・買掛金管理 | 本システムは売掛金管理を対象とし、買掛金管理は対象外とする。          |

---

## 1.5 想定利用者

本システムの想定利用者は以下とする。

| 利用者区分   | 主な利用内容                           |
| ------- | -------------------------------- |
| 経理担当者   | 請求書管理、入金データ取込、入金消込、売掛残高確認、月次締め確認 |
| 営業担当者   | 担当取引先の請求状況、入金状況、未入金状況の確認         |
| 営業管理者   | 部門別・担当者別の売掛金状況、滞留債権状況の確認         |
| 経理管理者   | 全社売掛残高、未消込入金、滞留債権、締め状況の確認        |
| システム管理者 | ユーザー管理、権限設定、マスタ管理、バッチ処理状況確認      |

---

## 1.6 システム化による期待効果

本システムの導入により、以下の効果を期待する。

| No. | 期待効果       | 内容                                     |
| --- | ---------- | -------------------------------------- |
| 1   | 消込作業時間の削減  | 入金データと請求データの自動照合により、手作業による消込時間を削減する。   |
| 2   | 未入金管理の強化   | 支払期限を超過した請求を迅速に把握し、回収対応を早期化する。         |
| 3   | 売掛残高の正確性向上 | 請求、入金、消込状況をリアルタイムまたは日次で反映し、残高精度を向上させる。 |
| 4   | 業務属人化の解消   | 担当者ごとのExcel管理を廃止し、標準化された業務プロセスを実現する。   |
| 5   | 経営判断の迅速化   | ダッシュボードにより、全社・部門別・取引先別の債権状況を即時に把握する。   |

---

## 1.7 基本方針

本システムの設計・開発における基本方針は以下とする。

| No. | 方針           | 内容                                          |
| --- | ------------ | ------------------------------------------- |
| 1   | Webシステムとして構築 | 利用者はブラウザからシステムを利用できるものとする。                  |
| 2   | マスタ管理を重視     | 取引先、請求先、支払条件、担当者などのマスタ情報を整備し、業務データの品質を確保する。 |
| 3   | 自動化と手動補正の両立  | 入金消込は自動化を基本とするが、照合できない場合は担当者が手動で補正できるものとする。 |
| 4   | 権限に応じた情報表示   | 利用者の役割に応じて、参照・登録・更新・承認・管理可能な範囲を制御する。        |
| 5   | 外部システム連携を考慮  | 銀行入金データ、会計システム、販売管理システムとの連携を考慮した設計とする。      |
| 6   | 監査性を確保       | 重要データの登録・更新・削除・消込処理について、操作履歴を保持する。          |

---

## 1.8 用語定義

| 用語    | 定義                                   |
| ----- | ------------------------------------ |
| 売掛金   | 商品・サービス提供後、取引先から将来回収予定の債権金額。         |
| 請求書   | 取引先に対して請求金額、支払期限、振込先等を通知する書類またはデータ。  |
| 入金    | 取引先から銀行口座等に振り込まれた金額。                 |
| 消込    | 請求データと入金データを照合し、債権が回収済であることを確定する処理。  |
| 未消込   | 入金データまたは請求データが存在するが、対応関係が確定していない状態。  |
| 滞留債権  | 支払期限を超過しても入金されていない売掛金。               |
| 支払条件  | 月末締め翌月末払い、20日締め翌月10日払い等、取引先ごとの支払ルール。 |
| 与信限度額 | 取引先に対して許容する売掛金残高の上限金額。               |
